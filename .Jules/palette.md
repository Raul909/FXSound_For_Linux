## 2024-05-15 - Add Keyboard Navigation to Custom Sliders
**Learning:** Custom interactive elements built with `<div>` (e.g., custom sliders like EQBand and EffectSlider) require significant manual work to be accessible. They need `role="slider"`, `tabIndex`, `aria-valuenow`/`aria-valuemin`/`aria-valuemax`, orientation logic, AND manual `keydown` event listeners to map Arrow keys, Home, and End to functionality. Otherwise, screen readers don't know what they are and keyboard users cannot interact with them.
**Action:** When creating any custom interactive component from standard `div` elements, proactively check W3C WAI-ARIA authoring practices for that role to ensure all necessary keyboard interactions and properties are implemented.
