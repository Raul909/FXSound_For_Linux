## 2024-05-16 - Accessible Custom Sliders
**Learning:** In the React frontend, custom UI components built with `<div>` (like the EQ and Effect sliders) completely lack native accessibility features, making them unusable for keyboard and screen reader users. Simply adding `tabIndex` is insufficient.
**Action:** Always implement full W3C-compliant slider patterns for custom range inputs: `role="slider"`, `tabIndex`, `aria-valuenow`/`min`/`max`, `aria-label`/`labelledby`, and keyboard event handlers (Arrows for steps, Home/End for boundaries) to ensure they are fully usable without a mouse. Add `:focus-visible` styles to provide clear visual focus indication.
