## 2024-03-06 - Improve accessibility for icon-only buttons and labeled selects
**Learning:** In React implementations where generic `<div>`s are used as visual labels next to interactive elements, screen readers will completely fail to associate them unless explicitly linked using `id` and `aria-labelledby`. Also, icon-only buttons that convey binary state (like power) must combine `aria-label` with `aria-pressed` to correctly indicate both function and current status to assistive technologies.
**Action:** When adding or reviewing custom interactive components (like dropdowns and status toggles), verify that visual text is explicitly linked to `<select>` or `<input>` tags using proper ARIA attributes, and ensure all purely visual icon buttons have corresponding textual alternatives and explicit state announcements.

## 2025-01-20 - Ensure W3C-compliant accessibility for custom slider components
**Learning:** Custom interactive slider components constructed with generic `<div>` elements natively lack essential accessibility features and keyboard interaction. Screen readers require explicit ARIA roles (`role="slider"`) and properties (`aria-valuenow`, `aria-valuemin`, `aria-valuemax`, `aria-label`) to convey value and purpose. Additionally, `tabIndex` must be used to make the component focusable, paired with comprehensive `onKeyDown` handlers to support standard W3C keyboard navigation (Arrow keys, Home, End), and explicit `:focus-visible` CSS rules to provide clear visual focus indicators.
**Action:** When creating or modifying custom sliders, always implement the `role="slider"` attribute, necessary ARIA value properties, keyboard navigation event handlers, and robust focus styling to ensure the components are fully usable by screen readers and keyboard-only users.
