## 2024-05-20 - Custom Slider Accessibility Requirements
**Learning:** Custom UI sliders built with standard `div` elements do not inherit native keyboard accessibility or screen reader semantics. Implementing full W3C-compliant slider accessibility requires explicitly mapping `role="slider"`, `tabIndex`, and `aria-valuenow`/`min`/`max` attributes to the exact container handling `onMouseDown`, and providing a matching `onKeyDown` handler that supports arrow keys, `PageUp`/`PageDown`, `Home`, `End`, and custom `Shift+Arrow` combinations for larger step sizes.
**Action:** When creating or modifying custom sliders, always ensure the container div serves as the interactive surface by adding the appropriate `role`, `aria-*` state properties, focus states (`:focus-visible`), and standard keyboard navigation handlers, bypassing the need for hidden native inputs if interaction models conflict.
