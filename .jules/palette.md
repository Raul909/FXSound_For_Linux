## 2024-03-09 - Custom UI Components and ARIA Attributes
**Learning:** In this application's custom design system, `<div>` elements are used as visual labels for `<select>` dropdowns instead of semantic `<label>` tags. Since they lack native association, these must be explicitly linked using dynamically generated `id` and `aria-labelledby` attributes (ensuring spaces in IDs are replaced with hyphens) for screen readers to announce them correctly. Also, icon-only toggle buttons must combine `aria-label` with `aria-pressed` to correctly convey state to screen readers.
**Action:** When working on custom UI elements or dropdowns in this repository, always explicitly map non-semantic visual labels to their form controls using ARIA attributes and ensure dynamic IDs are URL/HTML-safe. Always include `aria-pressed` on custom toggle buttons.
